## Slide Deck Workflow (NotebookLM → PPTX → Slidev)

The slide decks are **not** hand-authored. They follow this pipeline:

```
*-workshop.md  →  NotebookLM  →  .pptx  →  convert-pptx.py  →  .slidev.md + images
```

### Steps

1. **Send `*-workshop.md` to NotebookLM** — The workshop guide is the source of truth content. Upload it to NotebookLM and use it to generate a visual slide deck (PPTX).
2. **Export PPTX from NotebookLM** — Download the generated `.pptx` file and place it in `source/pptx/` (gitignored, not committed).
3. **Run the conversion script** — This extracts slide images and generates the `.slidev.md` file:

   ```bash
   npm run convert:pptx -- <workshop-folder-name>
   ```

   Example:

   ```bash
   npm run convert:pptx -- copilot-dev-agentic
   ```

4. **Output** — The script produces:
   - `public/images/<workshop>/slide-NN-HASH.png` — Full-bleed slide images (committed)
   - `workshops/<workshop>/<workshop>.slidev.md` — Slidev deck with `layout: image-full` and `background:` frontmatter (committed)

5. **Add presenter notes** — After conversion, manually add `<!-- Presenter notes -->` comments to each slide in the `.slidev.md` file. These are the talk-track guidance for the instructor.

### Key Points

- The `.pptx` files are **gitignored** — only the extracted images and `.slidev.md` are committed
- Re-running the conversion is **idempotent** — it overwrites previous output
- The `workshop.md` must be finalized before sending to NotebookLM (content changes after conversion require re-running the pipeline)
- **NotebookLM is ONLY used for the slide deck** — it does not generate the LAB or QUIZ files
- Labs and quizzes are authored separately following the established format in `copilot-dev-foundations`
